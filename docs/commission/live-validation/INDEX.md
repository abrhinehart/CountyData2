# Commission Radar — Live Validation Index

This directory holds per-jurisdiction recon notes from live scraper smoke-tests
against real agenda portals. Each note records: adapter family, portal URL,
date window, listing count, a 3-listing sample, a download smoke (size +
magic bytes), and any warnings. Notes are generated by
`scripts/cr_live_validate.py` (or a successor harness) against
`modules/commission/config/jurisdictions/<STATE>/<slug>.yaml`.

**No database writes. No Claude-extraction calls. Fetch + single download only.**

Passing this check means the adapter can reach the portal, parse listings for
a recent 6-month window, and pull at least one valid document. It does NOT
mean the jurisdiction is wired up for full extraction / matching — that is a
separate downstream step.
