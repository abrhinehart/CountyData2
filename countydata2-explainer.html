-- Count transactions by county
SELECT county, COUNT(*) FROM transactions GROUP BY county ORDER BY COUNT(*) DESC;

-- Recent builder purchases
SELECT grantee, subdivision, date, price
FROM transactions
WHERE type = 'Builder Purchase' AND date >= '2025-01-01'
ORDER BY date DESC LIMIT 20;

-- Transactions missing price (deed queue candidates)
SELECT county, COUNT(*) FROM transactions WHERE price IS NULL GROUP BY county;

Design Decisions

Architecture Insights

Deduplication via Generated Columns

PostgreSQL generated columns create normalized keys (uppercased, trimmed) from grantor, grantee, instrument, date, and county. A UNIQUE constraint prevents duplicates regardless of whitespace or casing variations in source data.

Multi-Lot Segmentation

A single deed can transfer lots across multiple subdivisions or phases. The transaction_segments table breaks these into individual rows with their own inventory_category and review_reasons, enabling per-segment analysis.

Lookup-Based Classification

Reference data (YAML files) is loaded into in-memory matchers at ETL start. SubdivisionMatcher uses longest-match-first substring search. Builder and LandBanker matchers use exact-match on normalized names. This avoids expensive DB round-trips per row.

Dual Legal Description Sources

The canonical shape separates export-sourced fields (export_legal_desc, export_legal_raw) from deed-sourced fields (deed_legal_desc, deed_legal_parsed). This preserves the original county export data while enriching with deed OCR extraction results.

Spatial-Ready (PostGIS)

PostgreSQL runs with PostGIS extension enabled. A geometry column is reserved for future parcel shapefile integration, geocoding, and spatial queries — enabling map-based analysis without a database migration.

Additive Migration Pattern

All 9 migrations are safe to re-run. They use IF NOT EXISTS for tables, ADD COLUMN IF NOT EXISTS for fields, and INSERT...ON CONFLICT for data. No destructive DDL — the schema only grows.

Help

Troubleshooting

Database connection refused ▼
Make sure Docker is running and the container is up: docker compose up -d. Check that your .env DATABASE_URL matches the docker-compose credentials (default: etl_user:changeme@localhost:5432/County-Data).
Encoding errors reading county files ▼
The reader tries 6 encodings automatically: utf-8, cp1252, latin1, iso-8859-1, utf-16, cp1250. If all fail, the file may be corrupted or use an unsupported encoding. Try opening in Excel and re-saving as UTF-8 CSV.
Subdivision not matching / showing as NULL ▼
The SubdivisionMatcher uses longest-match-first substring search against reference_data/subdivisions.yaml. If a subdivision isn't matching, either: (1) it's not in the YAML — add an entry with aliases, or (2) the legal description text differs from all known aliases. Run python seed_reference_data.py after adding new entries.
Migration fails or schema is out of date ▼
Run python apply_migrations.py — all migrations are idempotent and safe to re-run. If schema.sql was never applied, run psql -f schema.sql first against your database. Check migration 003 specifically — it's a Python backfill that requires the lookup matchers to be importable.
Bay price extraction not working ▼
The Bay price extractor uses Selenium with Chrome. Ensure Chrome is installed and webdriver-manager can download the matching driver. Try a dry run first: python bay_price_extract.py --limit 5 (without --apply) to test browser automation before writing to DB.
Raw-land benchmark OCR quality is poor ▼
Requires Tesseract OCR and pdf2image. The benchmark tool supports a --mode hybrid flag that uses OCR first, then falls back to Claude vision API for difficult pages. Ensure ANTHROPIC_API_KEY is set in your .env for the Claude extraction path.
Duplicate rows appearing in exports ▼
The dedup key is (grantor_key, grantee_key, instrument_key, date, county_key). If "duplicates" appear, the source rows likely differ on one of these fields — e.g., slightly different party names or instrument numbers. Check the source Excel file for subtle whitespace or casing differences.
